# Authentication & Password Reset Guide

## Backend Endpoints

### Register

`POST /api/auth/register`

- Body: `{ username, email, password, role }`
- Roles allowed: `admin`, `staff`, `manager`
- Returns newly created user (without password)

### Login

`POST /api/auth/login`

- Body: `{ username, password }`
- Returns JWT token and user info

### Forgot Password

`POST /api/auth/forgot-password`

- Body: `{ email }`
- If email exists, a reset link is sent using configured SMTP settings.
- Response always returns success message to avoid enumeration.

### Reset Password

`POST /api/auth/reset-password`

- Body: `{ token, newPassword }`
- Token must match the one generated by `/forgot-password` and not be expired (1h)
- New password must be at least 6 characters

## Database Changes

Migration `add-reset-password-fields-to-users-table.js` adds
`reset_password_token` and `reset_password_expires` to the `users` table.

Run it after pulling latest backend:

```bash
node database/migrations/add-reset-password-fields-to-users-table.js
```

## Frontend Pages

The React app now includes the following screens:

- `/register` – new user registration
- `/forgot-password` – request a reset link
- `/reset-password?token=...` – set a new password

### Testing the Flow

1. Start backend (`node server.js`) and frontend (`npm run dev`).
2. Visit `http://localhost:5173/register` to create an account.
3. Use `http://localhost:5173/forgot-password` and enter your email.
   - If SMTP is configured, you'll receive a link; otherwise the app shows a generic message.
4. Open the link (or copy token) to reach the reset page, choose a new password.
5. Login with new credentials at `/login`.

## Notes

- The reset email body simply contains a clickable URL; you may customize it.
- For development/testing without email, you can inspect the `users` table to read the
  `reset_password_token` value and manually navigate.
- Links assume frontend port `5173`; adjust if running elsewhere.
